# Airesto — Страница просмотра бронирований

Тестовое задание: страница просмотра бронирований для ресторана с интерактивной таблицей.

🔗 **[Live Demo](https://airesto-test-task-6q1irjcej-ledneevs-projects.vercel.app/)**

---

## Стек

- **Vue 3** (Composition API, `<script setup>`)
- **TypeScript**
- **Vite**
- **Pinia** — управление состоянием
- Без UI-библиотек — всё написано с нуля

---

## Реализованный функционал

### Основные требования
- Интерактивная таблица бронирований — столы по горизонтали, время по вертикали
- Получение данных из моковых данных с точным соответствием структуре API
- Переключение дат для просмотра бронирований
- Включение и отключение зон отображения (1 этаж, 2 этаж, Банкетный зал)
- Фиксированные заголовки — столы закреплены слева, время сверху при прокрутке
- Отображение текущего времени ресторана (с учётом таймзоны) красной линией
- Все типы событий из раздела "Возможные события": заказы (New, Bill, Closed, Banquet) и бронирования (Живая очередь, Новая, Заявка, Открыт, Закрыт)
- Правила расположения событий: наложение со смещением 4px и пересечение в колонки при разнице начал < 30 минут
- Hover-эффект с `backdrop-filter: blur` — размытие контента за активной карточкой
- Страница работает без тормозов на мобильных устройствах

### Дополнительные задания
- **Тёмная / светлая тема** — переключение в хедере
- **Скрытие контента при наложениях** — текст перекрытых карточек скрывается чтобы не было визуального шума
- **Выделение времени и столов** — drag-выделение как в Excel, работает на touch-устройствах. При отпускании показывает попап с деталями (дата, время, длительность, столы, вместимость) и кнопками "Создать" / "Отменить"
- **При нажатии "Создать"** — выводит в консоль ID столов, время начала и окончания, сохраняет событие локально
- **Поиск по имени** — поле в хедере с автодополнением, при выборе таблица скроллится к нужной карточке и подсвечивает её

### Архитектурные решения
- Моковые данные полностью соответствуют структуре swagger — для перехода на реальный API достаточно заменить `mockData` на fetch-вызов в store, остальной код не меняется
- Логика позиционирования событий вынесена в `utils/layout.ts` — компоненты не знают о математике расчётов
- Утилиты времени в `utils/time.ts` — работа с таймзонами, конвертация времени, генерация слотов

## Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/ledneev/airesto-test-task.git
cd airesto-test-task

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`

```bash
# Сборка для продакшена
npm run build
```

## Структура проекта

```
src/
  components/
    BookingTable/
      BookingTable.vue      # Основная таблица
      EventCard.vue         # Карточка события
      SelectionOverlay.vue  # Оверлей выделения
    DatePicker.vue          # Выбор даты
    ZoneFilter.vue          # Фильтр зон
    TheHeader.vue           # Шапка с поиском и переключением темы
    IconSvg.vue             # SVG иконки
  stores/
    bookingStore.ts         # Глобальное состояние
  types/
    index.ts                # TypeScript типы
  utils/
    time.ts                 # Утилиты работы со временем
    layout.ts               # Логика позиционирования событий
    overlapDetection.ts     # Логика обработки скрытия
    touchSelection.ts       # Логика обработки для touch устройств
  data/
    mockData.ts             # Моковые данные (заменяется на API)
```
